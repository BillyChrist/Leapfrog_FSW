Initializes the U\+S\+A\+R\+T2 peripheral for U\+A\+RT communication.

This function configures U\+S\+A\+R\+T2 with the following parameters for serial communication\+:


\begin{DoxyItemize}
\item {\bfseries{Baud Rate}}\+: Set to 115200 bps, which is commonly used for high-\/speed data transmission.
\item {\bfseries{Word Length}}\+: Configured to 8 bits (U\+A\+R\+T\+\_\+\+W\+O\+R\+D\+L\+E\+N\+G\+T\+H\+\_\+8B), allowing for standard character transmission.
\item {\bfseries{Stop Bits}}\+: Set to 1 stop bit (U\+A\+R\+T\+\_\+\+S\+T\+O\+P\+B\+I\+T\+S\+\_\+1), which is adequate for most communication protocols.
\item {\bfseries{Parity}}\+: No parity checking (U\+A\+R\+T\+\_\+\+P\+A\+R\+I\+T\+Y\+\_\+\+N\+O\+NE), ensuring that each byte is sent and received as a distinct unit.
\item {\bfseries{Mode}}\+: Both transmission and reception are enabled (U\+A\+R\+T\+\_\+\+M\+O\+D\+E\+\_\+\+T\+X\+\_\+\+RX), facilitating bidirectional data flow.
\item {\bfseries{Hardware Flow Control}}\+: Disabled (U\+A\+R\+T\+\_\+\+H\+W\+C\+O\+N\+T\+R\+O\+L\+\_\+\+N\+O\+NE), simplifying the communication without the need for additional control lines.
\item {\bfseries{Oversampling}}\+: Configured to 16 (U\+A\+R\+T\+\_\+\+O\+V\+E\+R\+S\+A\+M\+P\+L\+I\+N\+G\+\_\+16), enhancing timing resolution and communication reliability.
\end{DoxyItemize}

If the initialization process fails, the function calls {\ttfamily \mbox{\hyperlink{main_8h_a1730ffe1e560465665eb47d9264826f9}{Error\+\_\+\+Handler()}}}, which typically handles errors by entering an error state or logging the issue.

This function should be invoked during the system initialization phase to ensure that U\+S\+A\+R\+T2 is properly configured and ready for data transmission and reception. Users can insert custom code in the designated U\+S\+ER C\+O\+DE sections before or after the U\+A\+RT setup as needed.
